Bitbucket

Bitbucket é uma ferramenta de hospedagem e colaboração baseada em Git, com ambiente é favorável para o compartilhamento de dados, criação e implantação de códigos e automatização de testes no melhor conceito de cloud computing e infraestruturas on premises. A marca BitBucket foi adquirida pela Atlassian em 2010, o que garante às suas ferramentas integração total com os demais serviços da empresa e ainda workflows do Trello, e outros produtos da família Atlassian, como Jira, Fisheye e Bamboo. O Bitbucket era anteriormente uma startup independente, fundada por Jesper Noehr em 2008. Em setembro de 2015, Atlassian renomeou seu produto Stash para Bitbucket Server. Em julho de 2016, o Bitbucket adicionou suporte para Git Large File Storage (LFS).

O Bitbucket oferece suporte aos seguintes recursos:

Puxe solicitações com revisão de código e comentários. Bitbucket Pipelines, um serviço de entrega contínua. Verificação em 2 etapas e verificação em duas etapas exigida. Lista de permissões de IP. Verificações de mesclagem. Pesquisa de código (Alpha). Git Large File Storage (LFS). Documentação, incluindo arquivos README renderizados automaticamente em uma variedade de formatos de arquivo semelhantes ao Markdown. Rastreamento de problemas. Wikis. Sites estáticos hospedados na nuvem Bitbucket: sites estáticos têm o domínio bitbucket.io em seu URL. Complementos e integrações. APIs REST para construir aplicativos de terceiros que podem usar qualquer linguagem de desenvolvimento. Snippets que permitem aos desenvolvedores compartilhar segmentos de código ou arquivos. Espelhamento inteligente.

O que é um sistema de controle de versão?

Um sistema de controle de versão ou VCS, também conhecido como controle de revisão ou sistema de controle de origem, é um utilitário de software que rastreia e gerencia alterações em um sistema de arquivos. Um VCS também oferece utilitários colaborativos para compartilhar e integrar essas mudanças no sistema de arquivos a outros usuários VCS. Ao operar no nível do sistema de arquivos, um VCS rastreará as ações de adição, exclusão e modificação aplicadas a arquivos e diretórios. Um repositório é um termo VCS que descreve quando VCS está rastreando um sistema de arquivos. No escopo de arquivos de código-fonte individuais, um VCS rastreará adições, exclusões e modificações das linhas de texto dentro desse arquivo. As opções populares de VCS da indústria de software incluem Git, Mercurial, SVN e preforce.

Git

O Git é um sistema de controle de versões gratuito e de código aberto, criado por Linus Torvalds lá atrás, em 2005. Diferente de outros sistemas de controle em versão centralizada, como SVN e CVS, o Git é distribuído: todo desenvolvedor tem o histórico completo de seu repositório de códigos no local. Isso faz a primeira cópia do repositório demorar, mas as operações seguintes, como commit, responsabilidade, comparação, mesclagem e log, ficam muito mais rápidas. O Git também tem excelente suporte para ramificar, mesclar e reescrever a história do repositório, resultando em muitas ferramentas e em fluxos de trabalho inovadores e eficientes. As solicitações pull são ferramentas tão populares que permitem às equipes colaborar nos branches do Git e analisar com eficiência o código umas das outras. O Git é o sistema de controle de versões mais usado no mundo hoje em dia e é considerado o padrão moderno para o desenvolvimento de software. Saiba mais sobre Git.

Esta é uma visão geral de como o Git funciona:

  1. Crie um "repositório" (projeto) com uma ferramenta de hospedagem git (como o Bitbucket)
  2. Copie (ou faça uma cópia genérica) do repositório na sua máquina local
  3. Adicione um arquivo ao repositório local e "commit" (salve) as alterações
  4. "Passe" suas alterações para o branch principal
  5. Faça uma alteração no seu arquivo com uma ferramenta de hospedagem git e dê commit
  6. Dê um "pull" nas alterações para a sua máquina local
  7. Crie um "branch" (versão), faça uma alteração e dê commit
  8. Abra uma "solicitação pull" (sugira alterações no branch principal)
  9. "Mescle" seu branch com o branch principal
